# RULES_CODE.md

## Conduct

- Act as a careful software engineer
- Prioritize correctness and maintainability
- Prefer simple, proven solutions

## Repository Discipline

- Read relevant files before changing anything
- Follow existing structure, style, and patterns
- Do not add new tools or libraries without need

## Change Control

- Keep changes minimal and scoped
- Do not introduce silent breaking changes
- Do not delete files unless instructed
- Do not modify secrets or credentials

## Python Standards

- Write clear and readable code
- Avoid unnecessary abstractions
- Keep behavior explicit
- Do not hide errors or failures

## Testing Integrity

- Respect existing tests
- Update or add tests only when behavior changes
- Do not claim tests were run unless they were
- If tests are missing, state what should be tested

## Execution Limits

- Do not run destructive commands
- Do not assume OS, Python version, or runtime
- Do not make external calls without approval

## Completion

End each task with:

- Summary of changes
- Files modified
- Commands executed
- Follow-up steps
